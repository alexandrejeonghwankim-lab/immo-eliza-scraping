IMMO ELIZA — DATA DICTIONARY (FOR SALE)
=======================================
Database : data/immoeliza.db   (SQLite)
Table    : properties          (one row per unique property)

Conventions
-----------
- Missing information is stored as NULL.
- Binary attributes are stored as 1 (yes/present) or 0 (no/absent).
- The dataset is restricted to residential houses and apartments FOR SALE.
- Surfaces are in square metres (m2); amounts are in euros (EUR).
- Optional fields (heat pump, solar panels, flood scores, cadastral income,
  co-ownership charges, ...) are only published on some listings; they are
  captured when present and NULL otherwise.

Columns
-------
COLUMN                        TYPE      DESCRIPTION

-- Identity & provenance --
property_id                   TEXT      Unique Immovlan listing reference (primary key, no duplicates).
url                           TEXT      Full URL of the property detail page on immovlan.be.
source                        TEXT      Data source. Always "ImmoVlan".
scrape_date                   TEXT      Date the listing was scraped (YYYY-MM-DD).
posting_date                  TEXT      Date the listing was posted online (YYYY-MM-DD); from page metadata.
html_path                     TEXT      Relative path to the gzipped raw HTML snapshot (to re-parse later).

-- Classification --
property_type                 TEXT      Immovlan sub-type (e.g. residence, villa, apartment, duplex, penthouse).
category                      TEXT      Normalized category derived from property_type: "house" or "apartment".
transaction_type              TEXT      Listing type. Always "for-sale" in this dataset.
new_construction              INTEGER   1 if part of a new construction project, else 0.

-- Price --
price                         INTEGER   Sale price in EUR. NULL if "price on request".
price_per_sqm                 REAL      CALCULATED (not scraped): price / livable_surface, in EUR per m2.
vat                           INTEGER   1 if the sale is subject to VAT, else 0.
cadastral_income              INTEGER   Official Belgian taxable cadastral income in EUR.

-- Location --
street                        TEXT      Street name, if published.
house_number                  TEXT      House / building number, if published.
postal_code                   INTEGER   Belgian 4-digit postal code.
locality                      TEXT      City or municipality.
province                      TEXT      Belgian province (enriched from the postal code).
region                        TEXT      Flanders, Wallonia or Brussels (enriched from the postal code).
latitude                      REAL      Geographic latitude in decimal degrees, if available.
longitude                     REAL      Geographic longitude in decimal degrees, if available.

-- Interior --
bedrooms                      INTEGER   Number of bedrooms.
bedroom_surfaces              TEXT      Surface of each bedroom in m2, ';'-separated (e.g. "13;16;14"), when available.
livable_surface               INTEGER   Total interior living surface in m2.
living_room_surface           INTEGER   Surface of the living room in m2.
kitchen_surface               INTEGER   Surface of the kitchen in m2.
kitchen_equipment             TEXT      Kitchen equipment level (e.g. Fully equipped, Semi equipped, Not installed).
bathrooms                     INTEGER   Number of bathrooms.
showers                       INTEGER   Number of showers.
toilets                       INTEGER   Number of toilets.
build_year                    INTEGER   Year the building was constructed.
currently_leased              INTEGER   1 if the property is currently rented out, else 0.
furnished                     INTEGER   1 if sold furnished, else 0.

-- Technical / energy --
heating_type                  TEXT      Heating system (e.g. Gas, Electric, Fuel oil).
glazing_type                  TEXT      Type of glazing / windows (e.g. Double glass).
heat_pump                     INTEGER   1 if a heat pump is present, else 0.
solar_panels                  INTEGER   1 if solar panels are present, else 0.
air_conditioning              INTEGER   1 if air conditioning is present, else 0.
primary_energy_consumption    INTEGER   Specific primary energy consumption in kWh/m2/year.
epc                           TEXT      EPC / PEB energy performance label (A++ to G).
electrical_certificate        INTEGER   1 if the electrical installation certificate complies, 0 if not.
facades                       INTEGER   Number of open / exposed facades.
facade_orientation            TEXT      Orientation of the front facade (e.g. North, South-East).

-- Outdoor & extras --
garden                        INTEGER   1 if a garden is present, else 0.
garden_surface                INTEGER   Surface of the garden in m2.
garden_orientation            TEXT      Orientation of the garden (e.g. South-east).
terrace                       INTEGER   1 if a terrace is present, else 0.
terrace_surface               INTEGER   Surface of the terrace in m2.
balcony                       INTEGER   1 if a balcony is present, else 0.
garage                        INTEGER   1 if a garage is included or available, else 0.
garages                       INTEGER   Number of garage units.
indoor_parking                INTEGER   Number of indoor parking spaces.
outdoor_parking               INTEGER   Number of outdoor parking spaces.
cellar                        INTEGER   1 if a cellar is included, else 0.
swimming_pool                 INTEGER   1 if a swimming pool is present, else 0.
elevator                      INTEGER   1 if the building has an elevator, else 0.
running_water                 INTEGER   1 if connected to running water, else 0.

-- Risk / availability --
flooding_area_type            TEXT      Type of flooding area (e.g. no flooding area).
demarcated_flooding_area      INTEGER   1 if in a demarcated flooding area, else 0.
flood_g_score                 TEXT      Flood risk score for the building (G-score).
flood_p_score                 TEXT      Flood risk score for the plot (P-score).
availability                  TEXT      Availability status (e.g. On contract, Immediately).
building_state                TEXT      State / condition of the property (e.g. Normal, To renovate, New).

-- House-specific --
land_surface                  INTEGER   Total land / plot surface in m2.
ground_depth                  INTEGER   Depth of the plot in m.
terrain_width                 INTEGER   Width of the plot at the roadside in m.
number_of_floors              INTEGER   Number of floors (in the house; for apartments: floors in the building).

-- Apartment-specific --
apartment_floor               INTEGER   Floor on which the apartment is located.
maintenance_cost              INTEGER   Monthly common building charges / maintenance cost in EUR.
co_ownership_charges          INTEGER   Shared co-ownership charges in EUR, if shown separately.

Linked dataset
--------------
The for-rent companion dataset (data/immoeliza_rent.db, documented in
data_dictionary_rent.txt) holds rental listings for the same areas and can be
joined on postal_code / locality to study rent levels alongside sale prices.
